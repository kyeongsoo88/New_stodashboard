# 대시보드 확장 가이드

## 현재 상태
- ✅ CSV 파일 생성 완료 (`dashboard-sample.csv`)
- ⚠️ 아직 코드에서 CSV를 읽어오지 않음 (하드코딩된 데이터 사용 중)

## 확장 가능한 구조로 변경 방법

### 방법 1: CSV 기반 구조 (권장) ✅
**장점:**
- CSV 파일에 새로운 행만 추가하면 자동으로 대시보드에 표시
- 코드 수정 불필요
- 엑셀에서 쉽게 데이터 관리

**구현 시 구조:**
1. CSV 파일에서 데이터 읽기
2. 데이터 키를 컴포넌트 props로 매핑
3. 새로운 카드 추가 시:
   - CSV에 `카드_[새카드명]_값`, `카드_[새카드명]_YOY` 등의 행 추가
   - 코드는 자동으로 인식하여 표시

### 방법 2: 설정 파일 기반 구조
**구조:**
- `dashboard-config.json`: 카드 설정 (타입, 위치, 상세 항목 등)
- `dashboard-data.csv`: 실제 데이터

**장점:**
- 카드 타입(실판매출, 영업이익 등)을 설정 파일로 관리
- 데이터와 설정 분리

## CSV 기반 구조 구현 시 예시

### 새로운 카드 추가 방법

**1. CSV에 데이터 추가:**
```csv
데이터키,2025-01,2025-02,...,2025-10
카드_새카드명_값,1000,1100,...,1200
카드_새카드명_YOY,105%,106%,...,108%
카드_새카드명_채널_항목1_값,500,550,...,600
카드_새카드명_채널_항목1_YOY,110%,111%,...,112%
```

**2. 코드 자동 처리:**
- CSV 파싱 시 `카드_새카드명_*` 패턴의 모든 키를 찾아서
- 자동으로 `DetailedMetricCard` 또는 `MetricCard`로 렌더링

## 권장 구현 순서

1. CSV 파서 함수 작성
2. 데이터 매핑 함수 작성 (CSV 키 → 컴포넌트 props)
3. 기존 하드코딩된 데이터를 CSV 기반으로 교체
4. 테스트

이렇게 하면 새로운 항목을 추가할 때 **코드 수정 없이 CSV 파일만 수정**하면 됩니다!
